## Автотест для конструктора календаря мероприятий 3snet

Этот репозиторий содержит пример автотеста на **JavaScript + Playwright** для страницы-конструктора календаря мероприятий:

- URL: `https://dev.3snet.info/eventswidget/`

Тесты используют `@playwright/test` и базируются на конфигурации `playwright.config.js`.

### Что проверяет автотест

- **Базовая загрузка страницы**:
  - открытие страницы `https://dev.3snet.info/eventswidget/` без ошибок;
  - наличие заголовка «Начните создавать свой календарь мероприятий!»;
  - отображение шагов мастера «Шаг 1»–«Шаг 4».

- **Фильтрация по тематике и странам**:
  - доступность блоков «Выберите тематику» и «Выберите страны»;
  - имена тематик (Affiliate, Blockchain, Development, Igaming, Internet Marketing, SEO, Финтех) присутствуют в DOM (даже если элементы частично скрыты).

- **Генерация превью и код вставки**:
  - доступность кнопки «Сгенерировать превью»;
  - после клика отображаются подсказка «Скопируйте сгенерированный код» и кнопка «Скопировать код», что говорит о готовности кода для вставки на сайт.

### Особенности и наблюдения

- Страница использует много динамического контента и кастомных UI-компонентов (селекты, чекбоксы, конструктор кода вставки).
- В тестах используются в основном **поиск по тексту** (`page.getByText`), так как верстка и дата-атрибуты могут меняться, а пользовательские тексты более стабильны.
- При изменении текстов интерфейса (локализация/копирайтинг) тесты могут начать падать — это ожидаемое поведение и сигнализирует о необходимости их актуализации.
 - Для организации кода используется **Page Object Model**: класс `MainPage` расположен в `src/pages/mainPage.js`, а сами тесты — в `tests/eventswidget.spec.js`.

### Как запустить тесты локально

1. **Клонировать репозиторий**

2. **Установить зависимости**:

```bash
npm install
```

3. **Клонировать репозиторий с GitHub**:

```bash
git clone git@github.com:DmitriyRonMan/test_for_3snet.git
cd test_for_3snet
```

4. **Установить браузеры Playwright** (однократно, если ещё не установлены):

```bash
npx playwright install
```

5. **Запустить тесты**:

```bash
npm test
```

Команда `npm test` запускает фреймворк `@playwright/test` с конфигурацией из `playwright.config.js`. Никакой дополнительной настройки не требуется.

При необходимости можно открыть HTML-отчёт Playwright после прогона:

```bash
npx playwright show-report
```
